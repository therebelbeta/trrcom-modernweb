Why Angular+Node?

Speed - Node, as opposed to apache, delivers content asynchronously, meaning it can deliver content simultaneously whether or not another process is waiting. This speeds up response times tenfold, and allows for much more complex applications. Here's an example of how it works:

Since Node doesn't have to spin up a new instance every time a request is made, it uses drastically less memory, and can handle thousands more requests at a time.

 Maintainability - Node + Angular separates the concerns of developers, allowing for easier readable and maintanable code. Everything is Javascript - With Node, there is no need for a "front-end" and a "server-side" developer. Simply hire one developer who can work with the full stack! Open-source & Community-driven - Node + Angular are supported by legions of enthusiasts, with forums, wikis, how-tos and documentation litering the interwebs.

What else can you work in?

While I prefer working in Angular+Node, sometimes it's not right for the job. This portfolio, for instance, is written in a more simple jQuery frameork, since a full MVC would have been overkill. I still used Node, however, for compiling and optimizing. Here are a few things I am also known for working in: Vue.js - A stripped down dual-data binding utility, just like angular but more agnostic. jQuery - The backbone of the internet.
